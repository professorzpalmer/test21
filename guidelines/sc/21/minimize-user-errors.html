Minimize user errors

A

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 13 and Pull Request 97. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

  • Where all possible inputs come from a discrete set of data values, a user can select from a choice of valid input values without manually entering the value.
  • Where numerical data needs to be manually entered, alternative separator characters are accepted.

Exceptions:

  • Where the set of discrete data values is larger than thirty-one valid values.
  • Where limiting input values to a set of discrete values interferes with the main purpose of the content.
  • Where alternative separator characters could alter the numeric value, such as substituting commas with periods.
